AI로 기획서 만들기

Claude로 기획서 작성, 어렵지 않아요! 😊
가이드를 따라 초기 셋팅을 진행하고, 프롬프트를 생성해서 바로 시작해 보세요.

프롬프트 실행 전 아래 순서로 Claude 프로젝트를 셋팅하면 일관된 품질의 기획서를 반복 생성할 수 있습니다.

1
📄 MD 파일 설정

아래 3개 파일을 Claude 프로젝트 루트에 배치하면 Claude가 일관된 방식으로 기획서를 작성합니다.

🧠
CLAUDE.md
Claude의 작업 행동 원칙 정의. 불필요한 변경 방지, 단순성 우선 원칙 등 실수 패턴을 사전 차단합니다. andrej-karpathy-skills의 CLAUDE.md를 사용하세요.
https://github.com/multica-ai/andrej-karpathy-skills/blob/main/CLAUDE.md
✍️
writing_rule.md
용어·문체·화면 명칭 통일 규칙. 여러 사람이 작업해도 같은 톤과 표현으로 기획서가 작성됩니다. 팀 상황에 맞게 수정하여 사용하세요.
📐
instruction.md
기획서 프레임 구조, 섹션별 작성 항목, 에러 케이스 문서화 기준 정의. Claude가 이 파일을 참조해 일관된 기획서 구조를 유지합니다.
📂 파일 배치 위치 Claude Code 프로젝트 루트에 배치하거나, Claude.ai Project의 "Project instructions"에 내용을 붙여넣으세요.
프로젝트_루트/CLAUDE.md  ·  프로젝트_루트/writing_rule.md  ·  프로젝트_루트/instruction.md
2
🤖 서브 에이전트 설치

에이전트 파일을 .claude/agents/ 폴더에 배치하면 Claude Code에서 서브 에이전트로 호출할 수 있습니다.

AGENT
리뷰어 에이전트 (reviewer.md)
생성된 기획서의 완성도를 검토합니다. 누락된 에러 케이스, 불명확한 동작 정의, 정책 문서와의 불일치를 자동으로 찾아냅니다. 작업 완료 후 /reviewer로 호출하세요.
⏳ 파일 준비 중 — 곧 제공될 예정입니다
3
💡 Claude Code 기획서 작업 팁
1
Project 단위로 컨텍스트 유지하기
Claude.ai의 Project 기능을 활용하면 CLAUDE.md, writing_rule.md, instruction.md를 Project Instructions에 등록해 매번 붙여넣지 않아도 됩니다. 서비스별로 Project를 나눠 관리하세요.
2
Slash Command로 반복 작업 단축하기
.claude/commands/ 폴더에 review.md, add-state.md 등을 만들어두면 /review, /add-state로 즉시 호출할 수 있습니다. 자주 쓰는 기획서 작업 패턴을 커맨드로 등록해두세요.
3
메뉴 단위로 나눠 작업 → 리뷰 루프 만들기
메뉴 단위로 나눠 요청하고 각 메뉴 완료 후 리뷰어 에이전트를 돌리세요. "화면 생성 → 노션 동기화 → 리뷰 → 누락 보완"을 한 사이클로 묶으면 뒷 메뉴로 갈수록 일관성이 유지됩니다.
4
스크린샷·레퍼런스 이미지 적극 활용하기
Claude Code는 이미지를 직접 이해합니다. 참고할 기존 화면 캡처나 경쟁사 레퍼런스를 함께 첨부하면 더 정확한 레이아웃으로 기획서를 생성합니다. 특히 복잡한 목록형 화면에서 효과적입니다.
5
수정 요청은 구체적인 위치와 함께 하기
"2번 화면의 에러 상태 어노테이션 3번 — 네트워크 오류 시 재시도 버튼 추가해줘" 처럼 화면명·번호·수정 내용을 함께 명시하면 Claude가 정확히 해당 부분만 수정합니다. 모호한 수정 요청은 의도치 않은 변경을 유발합니다.

기획서 수정 작업을 위한 가이드입니다. 서비스 연결 후 아래 항목을 참고해 수정을 진행하세요.

4
메뉴별 정책 동기화
권장
각 메뉴 기획서 생성 후 Claude가 해당 화면의 정책을 아래 노션 페이지에 자동으로 기록합니다. 다음 메뉴 작업 시 이 페이지를 먼저 읽어 용어·조건·어노테이션 번호의 일관성을 유지합니다.
🗂
※ 3번 읽기용 정책 문서와 별도로, 기획서 작성 결과를 축적하는 전용 페이지입니다.
※ Notion MCP에 해당 페이지의 편집(Edit) 권한이 있어야 합니다.
① Claude Code — /mcp 로 연결
Claude Code 터미널에서 /mcp 를 입력하면 MCP 서버 목록이 표시됩니다. 목록에서 Notion을 선택해 연결하세요.
/mcp
② Claude.ai (Cowork) — 통합 설정에서 연결
Claude.ai → 설정 → 통합(Integrations) → Notion 추가 → 허용 범위 설정.
정책 동기화 기능을 사용한다면 편집(Edit) 권한을 반드시 허용해야 합니다.
③ 동기화 페이지 권한 확인
동기화 대상 노션 페이지에 Notion MCP 봇을 편집자(Editor)로 추가해야 합니다.
페이지 공유 설정 → "연결 추가" → Notion MCP 봇 검색 후 편집자 권한 부여.
⚠ 읽기 전용 문서 vs 동기화 페이지
3번 섹션의 "노션 정책 문서"는 읽기 전용이어도 됩니다.
이 섹션의 "정책 동기화" 페이지만 편집 권한이 필요합니다.

기획서 출력 설정
표준 — 엣지 케이스·에러 상태·빈 화면 등 모든 분기 처리 포함
🌐 HTML 파일로 기획서를 생성합니다
Figma 대신 브라우저에서 바로 열 수 있는 HTML 파일로 와이어프레임과 상세 디스크립션을 만들어 드립니다.
별도 MCP 연결 없이 Claude가 직접 파일을 생성합니다.
생성되는 파일 구성
메뉴마다 {프로젝트명}_{메뉴명}.html 파일이 생성됩니다.
📐 와이어프레임 — HTML/CSS로 구현한 실제 UI 레이아웃 (그레이스케일) 🔀 상태별 화면 — 기본 / 에러 / 빈 화면을 탭 또는 섹션으로 구분 📝 기능 어노테이션 — 번호 레이블 + 설명 테이블 📋 정책 메모 — 노션 연결 시 관련 정책 항목 포함
📝 기능 어노테이션 표시 방식
화면 하단에 어노테이션 패널을 고정 배치합니다. 모든 항목을 한눈에 볼 수 있습니다.
입력한 도구 이름으로 Claude가 설치 방법을 직접 검색해 연결한 뒤 기획서를 그립니다.
⚠ 프롬프트 입력 전 MCP를 먼저 연결해 주세요
Claude에 Figma MCP가 연결되어 있어야 정상 작동합니다.
아래 방법으로 설치 후 프롬프트를 붙여넣으세요.
🖼 Figma MCP 설치
Claude에서 /mcp 명령어를 입력하고 Figma를 검색해 설치하세요.
디자인 시스템 컴포넌트를 사용하려면 UI MCP를 선택하세요.
⚠ 프롬프트 입력 전 NXUI MCP를 먼저 연결해 주세요
아래 설치 가이드를 참고해 연결 완료 후 프롬프트를 붙여넣으세요.
① NXUI MCP 실행
생성된 프롬프트에 연결 지시가 자동 포함됩니다. Claude가 아래 명령어로 MCP를 실행합니다.
npx -y @nxui/mcp
② NXUI 패키지 설치 (프로젝트에 미설치 시)
프로젝트 루트에 .npmrc 파일을 만들고 아래 내용을 추가하세요.
@nxui:registry=https://nexus.nexon.com/repository/nxui-npm-release/
// npm
npm install @nxui/core

// yarn
yarn add @nxui/core

// pnpm
pnpm install @nxui/core
③ 스타일 설정
프로젝트의 root CSS 파일에 추가하세요.
@layer reset, core, tailwind;

@layer reset {
  @import '@nxui/core/reset.css';
}

@layer core {
  @import '@nxui/core/styles.css';
}

@layer tailwind {
  @import 'tailwindcss/theme.css';
  @import 'tailwindcss/utilities.css';
}
위 가이드를 Claude에 전달해 설치를 자동으로 진행시킬 수 있어요.
입력한 UI MCP 이름으로 Claude가 설치 방법을 검색해 연결한 뒤 컴포넌트를 활용합니다.

Figma MCP 전용 설정입니다.

피그마 위치 지정
선택사항
피그마에서 파일을 열고, 브라우저 주소창의 URL을 그대로 복사해 붙여넣으세요.
💡 좌표 찾는 법 — 피그마 캔버스에서 빈 곳을 클릭하면 오른쪽 패널 상단에 X, Y 좌표가 표시돼요.
새 기획서를 기존 프레임 바로 옆에 붙이고 싶다면, 그 프레임의 오른쪽 끝 X값에 80 정도를 더한 값을 입력하면 됩니다.
비워두면 Claude가 빈 공간을 찾아 자동 배치합니다.
위치를 따로 지정하지 않으면 Claude가 메뉴 수에 맞게 자동으로 레이아웃을 잡아 새 섹션에 배치합니다. 기존 파일에 추가할 때만 위치를 지정해 주세요.
